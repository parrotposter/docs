# Введение

Добро пожаловать в документацию GraphQL API сервиса ParrotPoster. Данное API позволяет автоматизировать публикацию постов в социальные сети, управлять вашими аккаунтами и интегрировать функционал ParrotPoster в ваши приложения и сайты.

## Общие сведения

API ParrotPoster построено на технологии **GraphQL**. Это означает, что вы можете запрашивать именно те данные, которые вам нужны, и получать их в одном запросе.

**Базовый URL для доступа к API:**

```text:no-line-numbers
https://parrotposter.com/api/graphql
```

API работает с двумя типами HTTP-запросов:

- **GET-запрос** открывает в браузере интерактивную среду разработки **GraphiQL** (иногда называемую "playground"). Это удобный способ изучать API, просматривать доступные методы (схему) и тестировать запросы вручную.
- **POST-запрос** предназначен для непосредственного выполнения GraphQL-запросов и мутаций из вашего кода. Все взаимодействие с API в production-среде должно происходить через POST-запросы.

[Перейти к Playground](https://parrotposter.com/api/graphql)

## Статус API и обратная совместимость

В настоящее время API находится в активной разработке, однако его текущая версия является стабильной. Мы стремимся к тому, чтобы все изменения были обратно совместимыми:

- Существующие методы и поля не будут удаляться без предварительного уведомления.
- Если поле объявляется устаревшим, ему будет присвоена директива `@deprecated` с указанием причины и альтернативы. Оно будет поддерживаться еще некоторое время, прежде чем будет окончательно удалено.
- Новая функциональность будет добавляться путем введения новых полей и методов, не затрагивая существующую логику.

Это означает, что ваша интеграция, однажды написанная под текущую версию, продолжит работать и в будущем.
