# Обновление поста

Мутация `updatePost` позволяет обновить пост, который еще не был опубликован, или который был опубликован в прошедший день.

::: warning Важно
Нельзя обновить пост, если прошли сутки с момента его публикации.
:::

## Структура входных данных

Мутация принимает объект `post` со следующей структурой:

```graphql
mutation {
  updatePost(
    id: "post_id"
    post: {
      fields: { ... }
      networks: { ... }
      publishAt: "2024-01-12T04:00:19.123Z"
    }
  ) {
    id
  }
}
```

## Параметр `id`

Уникальный идентификатор поста, который нужно обновить.

## Параметры `post`

Данные структуры `post` идентичны в мутации [createPost](./create.md).

## Ответ `updatePost`

Мутация возвращает объект обновленного поста. Вы можете запросить любые поля поста в зависимости от ваших потребностей. Посмотрите [структуру объекта Post](./item.md).

## Важные замечания

1. Обновление картинок и ссылок в уже опубликованном посте не будут применены в некоторых соцсетях.
2. Нельзя менять дату публикации уже опубликованного поста.
